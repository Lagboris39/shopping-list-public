# Gemini plan - カード並び替えバグ修正の現状と今後

カード並び替えに関する一連の修正について、現状のステータスと解決すべき課題をまとめました。

## 現在の状態 (v5 実装直後)

### 成功している修正
- [x] **並び替えマージロジックの修正**: カテゴリ表示中に並び替えても、他のカテゴリのアイテムが飛んだり順序が壊れたりしない（`handleReorder` の改善）。
- [x] **ドラッグ操作感の改善**: CSSの `transition: transform` による「遅れ」や「カクつき」を排除。
- [x] **タップフィードバックの広域化**: ハンドルだけでなくカード全体を触った際に暗転（`brightness` + `inset shadow`）するよう改善。
- [x] **色の固まりバグの解消**: `onDragEnd` や `onPointerUp` など、あらゆる終了ポイントで React state (`isPressing`) をリセット。

### 未解決の課題
- [ ] **待機時間中の「吸い付き」不足**: 400msの待機中に指が動いた分、ドラッグ開始時にカードと指の位置がずれてしまう。
    - 原因: `dragControls.start(latestEvent)` を呼んでも、framer-motion 内部で「最初の pointerdown」または「要素の初期位置」を基準としたオフセット計算が残っている可能性がある。

---

## 検討中の修正方針 (v6 案)

「待機時間中に動いた指の位置」へカードを吸い付かせるためのアプローチ：

### 案1: ドラッグ開始時に要素を物理的に移動（ワープ）させる
`dragControls.start(latestEvent)` を呼ぶ直前に、待機中に移動した距離分だけカードの中身（`motion.div`）を `x` / `y` プロパティで瞬時に移動させる。
- メリット: 直感的。
- デメリット: スワイプ用の `x` (useMotionValue) と競合する可能性がある。

### 案2: 合成 PointerEvent の再設計
前回失敗した合成イベントをより正確に（`isPrimary`, `pointerType` などのプロパティを模倣して）作成し、framer-motion に「これが最初のタッチダウンだ」と信じ込ませる。

### 案3: 待機時間を短縮、または条件付きで即時開始
8px以上の移動を検知した時点で（400ms待たずに）ドラッグを開始させる。ただし、スワイプ操作（左右）やスクロール（上下）との判別をより厳密にする必要がある。
